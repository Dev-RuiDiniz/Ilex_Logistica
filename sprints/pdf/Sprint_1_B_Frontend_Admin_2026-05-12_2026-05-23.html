Checklist diario Scrum

  • Atualizar status da tarefa no board (Todo, In Progress, Done).
  • Registrar impedimentos e responsavel pela remocao.
  • Confirmar evidencia da tarefa (teste, log, link de issue/PR).
  • Definir proxima acao objetiva para cada tarefa em andamento.

Definition of Done (resumo)

  • B-01: App inicia localmente com rota raiz funcional.
  • B-02: Shell aplicável às páginas protegidas e responsivo.
  • B-03: Login funcional com feedback de erro/sucesso e redirecionamento.
  • B-04: Navegação protegida consistente para usuários não autenticados.
  • B-05: Interface respeita matriz de perfis definida.
  • B-06: Listagem integra com API e trata loading/erro.
  • B-07: Fluxo de create/update com retorno visual de sucesso/erro.
  • B-08: Inativação protegida e sincronizada com API.
  • B-09: Documento revisado por frontend e backend.
  • B-10: Roteiro executável com evidências esperadas por caso.

Criterios de aceite testaveis

  • B-01: npm run dev
  • B-02: Usuário autenticado vê menu e conteúdo em desktop/tablet sem quebra.
  • B-03: Credencial válida autentica; inválida exibe erro claro sem quebrar UI.
  • B-04: Acesso anônimo a rota privada redireciona para login.
  • B-05: Perfil auditoria sem ações de edição; admin com acesso completo.
  • B-06: Filtro por nome funciona; status ativo/inativo é exibido corretamente.
  • B-07: Criar e editar reflete na tabela; validação bloqueia envio inválido.
  • B-08: Após confirmação, item some da visão padrão sem recarregar página.
  • B-09: Fluxo completo descrito com critérios de acesso por perfil.
  • B-10: Casos felizes e de erro principais documentados com resultado esperado.
