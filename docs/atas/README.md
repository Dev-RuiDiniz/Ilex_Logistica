# Atas e templates de cerimônias Scrum

## Objetivo

Centralizar os templates da Sprint 1 para registrar decisões, impedimentos, riscos, evidências e ações com responsável e prazo.

## Templates disponíveis

| Cerimônia | Arquivo | Uso |
|---|---|---|
| Sprint Planning | `planning.md` | Definir objetivo, backlog, capacidade, riscos e decisões iniciais. |
| Daily Scrum | `daily-template.md` | Acompanhar progresso diário, impedimentos, riscos e próximas ações. |
| Sprint Review | `review.md` | Validar entregas, evidências e critérios de aceite com base no plano QA. |
| Sprint Retrospective | `retro.md` | Registrar aprendizados, melhorias e ações objetivas para a próxima sprint. |

## Checklist de rastreabilidade

- [ ] Toda decisão tem motivo, impacto e responsável.
- [ ] Todo impedimento tem ação, responsável, prazo e status.
- [ ] Toda melhoria tem forma de medição.
- [ ] Toda ação tem responsável e prazo.
- [ ] Review referencia `qa/plano-s1.md`.
- [ ] Riscos P0/P1 são acompanhados a partir de `sprints/riscos-s1.md`.
