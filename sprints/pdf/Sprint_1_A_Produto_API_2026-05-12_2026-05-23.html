Criterios de aceite testaveis

  • A-01: GET /health
  • A-02: API conecta no banco local; erro de credencial retorna falha tratável e logada.
  • A-03: Tabelas de usuários/perfis criadas com constraints básicas válidas.
  • A-04: Login válido retorna token; inválido retorna 401; token expirado é recusado.
  • A-05: Usuário sem perfil recebe 403; perfil auditoria mantém leitura sem edição.
  • A-06: Criação e edição persistem; inativação remove da listagem padrão.
  • A-07: Payload inválido retorna 422 padronizado; respostas válidas seguem schema.
  • A-08: Casos críticos de auth e carriers passam sem falha.
  • A-09: Inclui exemplos de request/response e regras de autorização por perfil.
  • A-10: Todos os critérios P0 mapeados com status, responsável e evidência.
