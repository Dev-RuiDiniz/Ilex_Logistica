# UAT formal do MVP

Estado: **preparado; execução e assinaturas pendentes**.

Cada perfil possui roteiro separado. Antes da sessão, preencher ambiente, URL, tag/build, data, executor, usuário sanitizado e dataset. Para cada cenário registrar `PASSOU`, `FALHOU` ou `BLOQUEADO`, evidência e defeito relacionado.

Defeito crítico/alto bloqueia go-live. Aceite de risco exige ID, responsável, justificativa, mitigação, prazo e assinaturas. Evidências não podem conter token, senha, dump, arquivo integral nem dado pessoal.
